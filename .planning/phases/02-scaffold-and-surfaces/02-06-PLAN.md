<objective>
Differentiate fail-loud vs graceful-empty terminal states in LiveBlockerPanel rendering. The worker's `flatten-blocker-chain` handler uses `terminal.kind = 'EXTERNAL'` as a catch-all for both error states (relations unavailable, startId required) AND for the graceful "no active blockers" case. The UI currently renders `{terminal.kind.replace(/_/g, ' ')}` verbatim, surfacing "EXTERNAL" in the user's face for every graceful-empty case. Fix: discriminate by terminal.label or add a new graceful-empty kind to the Terminal type.

Output: Reader's right rail shows nothing or a green "No active blockers" badge for the graceful case; operators still see fail-loud terminal text if it ever surfaces (which Plan 02-03c made structurally impossible but defense-in-depth).
</objective>

UI currently treats EXTERNAL as a single rendering path, surfacing the kind label. Fix: branch on terminal.label inside the EXTERNAL kind, OR add a new terminal.kind value 'NO_BLOCKERS' / 'UNAVAILABLE' so UI can dispatch cleanly.
