<objective>
Wave 2 — the chat-topic lifecycle helpers + the per-send wake nudge + the topic-description "converse only" instruction. This plan implements the D-09 + D-11 + D-12 trio that, together, makes multi-turn chat reliable: the chat-topic issue is held non-terminal (D-09), a watchdog flips it back if observed terminal/blocked (D-11), and an explicit per-send `requestWakeup` nudge ensures the agent re-wakes on every operator message (D-12).

Purpose: Phase 4's chat-topic issue is a real Paperclip issue. When an agent runs the full task lifecycle on it, post-completion machinery (disposition / recovery owner / `finish_successful_run_handoff`) engages and either (a) marks the topic `done` (terminal → un-wakeable per `TERMINAL_ISSUE_STATUSES` in `issue-tree-control.ts`) or (b) parks it in `blocked` (the recovery escalation target). Either way, multi-turn breaks. This plan's helpers prevent the prevention (description tells the agent to stay non-terminal) AND backstop deterministically (worker flips the status back + nudges the wake) AND surface the residual host-blocked case as a signal Plan 04.1-04 wires into the host-stuck banner.
