## Decisions Made

See `key-decisions` in the frontmatter. The two load-bearing ones:

1. **Migration 0007 is the steady-state path, not a fallback.** Wave 1 OQ-2 verdict made this lock unambiguous: REST originId filters return 500-row cap or 0 rows regardless. The side table is the steady-state D-08 path; the plan's "fallback" framing is superseded.

Best-effort symmetry with the marker-comment write keeps the back-link write right next to the create, where it belongs — and if a host RPC blip causes both to miss together, the operator sees both effects together (no marker + no rail entry).
