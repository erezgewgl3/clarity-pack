## Accomplishments

Four surgical UI fixes that close every Plan 04.1-09 drill finding without breaking the worker-tier contracts established in Plans 04.1-02..04.1-05:

1. **Inline task card + creation toast (drill fix #1, CRITICAL).** `onDialogSuccess` in `src/ui/surfaces/chat/index.tsx` now does FOUR things (was two): closes the dialog, bumps refreshKey, sets `pendingTaskCard = { issueId, title }` for promote mode so the inline card lights up immediately, and fires `showToast({ message: '↗ Task created — <short-id>, assigned to <employee>.', duration: 6000 })` for BOTH promote and cold modes. The optimistic `pendingTaskCard` is cleared when the chat.messages poll catches up to the marker comment / on topic switch / on new send. Cold tasks get a toast but no inline card (cold tasks aren't topic-anchored, by spec). 6 grep hits for `setPendingTaskCard` + `Task created` in index.tsx.

2. **Dialog DETAILS textarea sizing (drill fix #2, HIGH).** `[data-clarity-surface="chat"] .true-task-dialog textarea` gets `min-height: 140px; max-height: 40vh; overflow-y: auto; resize: vertical;`. Operators authoring real task bodies now have room to type; the textarea caps at 40vh so the dialog stays comfortably on screen; vertical resize lets power users drag larger.

3. **Horizontal viewport overflow fix (drill fix #3, HIGH).** The chat shell's `.thread` grid item refused to shrink below intrinsic content width — adding `min-width: 0` lets it shrink. `.topics` container gets `min-width: 0; max-width: 100%; overflow-x: auto;` so the topic strip scrolls horizontally inside its lane. `.topic` chips cap at 280px with `flex-shrink: 0`; `.topic-title` ellipsis-truncates at 220px. Full title surfaces on hover via the existing `title={topic.title}` HTML attribute. At viewport 1280px+, `document.documentElement.scrollWidth === clientWidth` is now true.

4. **Resume heartbeat inline toggle (drill fix #4, MEDIUM).** When `ceoStatus === 'paused'`, the Quick Action row renders `▶ Resume heartbeat` instead of `⏸ Pause heartbeat`. Click invokes `ctx.actions.invoke('agents.resumeHeartbeat', ...)`; on success, toast `Heartbeat resumed for <employee>.` fires + status pill flips back to `live` optimistically. If the host doesn't expose `agents.resumeHeartbeat` yet, the toast says `... (host call pending — verify on the agent page).` AND the visual flip still happens (graceful degrade). Operator no longer has to navigate to the agent page to resume.

## Decisions Made

See `key-decisions` in the frontmatter. The two load-bearing ones:

1. **Resume RPC is best-effort, never required.** `ctx.actions.invoke('agents.resumeHeartbeat', ...)` is attempted; on failure (action not exposed yet on this host), the toast still fires with a hint AND the optimistic visual flip still happens. This unblocks 4.1 closure without waiting on a Paperclip host upstream change. Future host PR to expose the action key tightens the hint toast (handled in 4.2 followup item 5).

2. **Cold task creation gets a toast but NO inline card.** Cold tasks aren't topic-anchored — no marker comment is posted to the current thread. The toast is the right + only operator confirmation for the cold path. This matches the cold-task semantics established in Plan 04.1-08 (cold tasks live in the Issues list, not in any chat thread).
