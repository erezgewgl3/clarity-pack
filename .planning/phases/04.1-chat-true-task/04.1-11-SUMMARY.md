## Accomplishments

Two surgical fixes bundled into one ship — both verified PASS on a single Countermoves drill:

1. **Fix A — Bulletin cadence Date-compare gate (Phase 3 cross-phase patch, `src/worker/jobs/compile-bulletin.ts`).** Extracted the gate decision into a pure exported helper `isPastDue(now: Date, nextDueAtIso: string): boolean` using Date comparison. The compile-bulletin per-company loop now reads `if (!isPastDue(now, nextDueAtIso)) continue;`. 2. **Fix B — Marker-pattern allowlist in classifyComment (Phase 4.1 surgical patch, `src/worker/chat/comment-classify.ts`).** Added a marker-pattern allowlist as the FIRST check in `classifyComment`, BEFORE the primary `authorType === 'system'` discriminator. The Plan 04.1-02 marker comment `Task created — <issueId>, assigned to <employeeName>.` ALWAYS classifies as `'conversation'` regardless of authorType. Tight regex `/^Task created — [^,]+, assigned to .+\.$/` matches the canonical marker shape only (em-dash, single non-comma chunk, single non-empty chunk, period terminator). Does NOT broaden the allowlist beyond this exact pattern. `RUNTIME_PHRASES` table and the existing primary/secondary discriminators are UNCHANGED. The Plan 04.1-04 explanatory comment block at lines 32-39 was rewritten to reflect the corrected understanding (Plan 04.1-04 assumed plugin-worker createComment calls would carry `authorType:'agent'` or `'user'`; production showed otherwise — Countermoves host stamps them `authorType:'system'`).
