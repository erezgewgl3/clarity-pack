## Lessons Learned

1. **String-compare timestamps with mixed formatters is ALWAYS a bug.** The Plan 04.1-11 bundled patch fixed a Phase 3 cadence gate that used `now.toISOString() < nextDueAtIso` -- a lexicographic compare that reverses chronology when only positions 10+ differ (ISO-T from JS vs space-separator from Postgres). Plan 03-10's closure drill passed because `next_due_at` had advanced to a different day; the same-day case was a latent bug uncovered later. Captured in memory `bulletin-cadence-gate-string-bug.md` and pinned by a regression-pinning assertion in `compile-bulletin-gate.test.mjs` that documents the BUG, not just the FIX.

2. **Drill scenarios must cover same-day cases.** The cadence gate above is a perfect example -- the cross-day fixture passed by accident. Future cron-gate drills must include same-day fixtures explicitly.

3. **Allowlist BEFORE filter.** When a known-safe set of rows should bypass a downstream filter, put the allowlist at the HEAD of the classifier, not as a special case inside the filter. The allowlist is a positive assertion ("this row IS conversation"); the filter is a negative one ("this row is NOT runtime-noise"). The Plan 04.1-11 marker allowlist is at line 1 of `classifyComment`; the dual-keyed discriminator from Plan 04.1-04 runs second.

4. **Plugin-worker createComment is stamped with authorType:'system' on this host.** Documented in the spike's PROBE-D14-DISCRIM finding (the disposition-recovery service stamps it that way too -- the same field, different writer). Future host-field-first discriminators must consider this collision and use either a marker-style allowlist OR a writer-identity check distinct from authorType.

5. **Cross-plan retrofit pattern.** Plan 04.1-05 added a side-table back-link write to `createTrueTask` -- a file in Plan 04.1-02's `files_modified` set. The retrofit was filed as `[Rule 3 -- Cross-plan retrofit per Wave 1 lock]` with a 4-test RED->GREEN, an optional context field to keep existing test fixtures working, and best-effort symmetry with the existing marker-comment discipline. Future plans that discover constraints requiring upstream-helper extensions should follow this template.

6. **Iterative-drill plan chain works for UI surfaces.** Plan 04.1-06 -> 04.1-08 -> 04.1-09 -> 04.1-10 was a 4-cycle iteration where each cycle landed its own RED->GREEN tests, version bump, and tarball. Operator could roll back exactly one step on a failed drill rather than untangling a layered patch. The lineage in this VERIFICATION.md documents the trail explicitly.

7. **Spike-locked drops as Wave 1 finding.** Plan 04.1-01 PROBE-OQ3 PASS-NATIVE made `ctx.issues.requestWakeup` unnecessary AND the host returned 404 anyway. Plan 04.1-03 dropped the call per the Wave 1 lock, pinned by a regression-guard test ("NEVER calls ctx.issues.requestWakeup") across 5 status paths. The pattern: a spike finding can OVERRIDE plan text if Wave 1 lock-down explicitly authorizes the override.
