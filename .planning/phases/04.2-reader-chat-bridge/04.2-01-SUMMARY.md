## Decisions Made

- **Migration 0009 ships one statement, not two.** The plan prose + acceptance grep expected an `ADD COLUMN IF NOT EXISTS` plus a partial `CREATE INDEX IF NOT EXISTS`. The host's DDL-prefix validator (`test/migrations/ddl-prefix-validator.test.mjs`) rejects a standalone `CREATE INDEX`. The migration ships only the column add. The reverse-topics SELECT is company-scoped and single-operator-scale, so the missing index is immaterial (threat T-04.2-01-05 was already `accept`-dispositioned).
- **ContinueInChatButton labels with `assigneeAgentId`,** not a friendly employee name — the Reader carries no roster and `chat.openForIssue` returns no display name.
- **Navigation uses `useHostNavigation` / `useHostLocation`** (the SCAF-09 repo convention), not react-router `useNavigate` / `useSearchParams` — there are no such hooks in this codebase.
