## What was built

**Plan 04.2-03 closes GAP-RCB-03-CARRIER and exposes GAP-RCB-03-DISPATCH one layer deeper.**

The Paperclip host wrapper around `useHostNavigation().navigate()` strips both `?query` (via `resolveHref`) and `{ state }` (before reaching react-router; `history.state.usr === null` on the live host) on cross-route navigation. Plan 04.2-01 emitted `?query` (stripped). Plan 04.2-02 swapped to `{ state }` (also stripped). Plan 04.2-03 swapped the carrier inside the existing `src/ui/surfaces/chat/deep-link.mjs` contract module to **URL hash (`#h=<base64-JSON>`)** — empirically proven on live Countermoves to survive end-to-end. The contract module's public API stayed shape-stable (consumers unchanged).

The live-host carrier verdict came from an operator-run browser probe: see `scripts/probes/carrier-survival.mjs` OPERATOR-OUTPUT block.

The Task 5 operator drill on Countermoves PASSED everything that the carrier swap is responsible for (install/uninstall, schema unchanged, row counts byte-identical, hash carrier survives end-to-end, chat-side consume effect runs and clears the fragment). But it failed Path 1 of the 5-path Reader↔Chat Bridge drill because the **dispatch step inside the chat surface** does not call `setEmployee` — exposing GAP-RCB-03-DISPATCH, a latent defect that 04.2-01 and 04.2-02 could never surface because no deep-link payload ever arrived to dispatch.
