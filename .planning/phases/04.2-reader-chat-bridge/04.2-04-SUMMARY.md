## What was built

**Plan 04.2-04 ships the dispatch fix for GAP-RCB-03-DISPATCH** — the latent defect in the chat surface's existing-topic deep-link path that Plan 04.2-03's URL_HASH carrier swap finally surfaced. The fix is surgical: lift `chat.roster` into `ChatPageBody`, look up the employee by UUID, call `setEmployee` with the matched `RosterEmployee`, thread `link.employee` into `setTopic`'s `employeeAgentId`, and gate the consume on roster availability so the deep link survives the typical race where it arrives before `chat.roster` resolves.

The dispatch defect lived in the codebase since Plan 04.2-01 was written (the `employeeAgentId: ''` hardcoded empty in the existing-topic dispatch). It was structurally invisible until 0.9.2's URL_HASH carrier finally let a payload through to dispatch.

    Comment explains that the host bridge's `usePluginData` deduplicates identical `(action, params)` pairs, so this parallel call shares the underlying fetch with `RosterRail`.

  - Replaced the hardcoded `employeeAgentId: ''` in `setTopic({...})` with `employeeAgentId: link.employee ?? ''`.
  - Extended the `useEffect`'s dep array: `[search, pathname, hash, locationState, nav, roster, rosterLoading]`.
