6. `new-topic-needed` cold-issue fallback (unchanged — D-11 baseline preserved)

## Forward defects filed to Plan 04.2-08

1. **GAP-D8-LINEAGE-TOOLTIP** — originId-lineage Continue tooltip uses raw topic-issue UUID instead of CHT-NN identifier. Worker handler `chat.openForIssue` returns `topicIssueId: <UUID>` for the lineage path; UI tooltip uses it directly. Fix: resolve CHT-NN identifier server-side and add `topicIdentifier?: string` to the result payload; update UI tooltip expression. Pre-existing at rc.6 — Plan 04.2-07 didn't introduce, but plan's success criterion 5 required "NO UUID-shaped strings for any path tested" which this path violates.
2. **GAP-D8-REVERSE-TOOLTIP-FALLBACK** — reverse-lookup Continue tooltip uses fallback `this issue` instead of BEAAA-NNN identifier. Either the worker handler isn't returning `sourceIssueIdentifier` on the reverse-lookup path (only on ambiguous), or the UI tooltip expression has a branch gap. Fix: thread `sourceIssueIdentifier` through both `existing-topic` (reverse-lookup) and `existing-topics-ambiguous` result branches; update UI tooltip expression.
3. **GAP-PICKER-ROW-DISPATCH** — picker row click navigates to chat surface but doesn't open the picked topic; user lands at "Select an employee" empty state. Picker row click emits topic-only URL_HASH via `buildTopicDeepLink(companyPrefix, topicIssueId)` (D-07 lock); chat-surface existing-topic dispatch requires `link.employee` for the roster-match → `setEmployee(matched)` chain. Fix options: (a) `buildTopicDeepLink` adds employee to URL_HASH (picker has employee_agent_id at row-render time per Task 4); (b) chat-surface existing-topic dispatch derives employee from topic when `link.employee` is undefined.
4. **GAP-RCB-05-CHIP-STYLING** — `About COU-NNNN →` backlink chip in topic-strip renders as a circle/oval with wrapped text instead of a rectangular chip. Visible on freshly-created topics. Pre-existing styling glitch surfaced more clearly during this drill.
