## Area: Resume policy + multi-topic disambig

### Turn 1 — Resolution algorithm

Question: *"Resume policy — when operator hits Continue-in-chat on a cold issue (not chat-spawned) that has existing chat topics referencing it via `origin_issue_id` (RCB-04), what should `chat.openForIssue` resolve to?"*

Options:
1. **Resume by assignee, picker only when ambiguous** *(recommended)* — lookup by `origin_issue_id` AND `employee_user_id=assignee`; 0/1/≥2 cardinality branching.
2. Resume any topic, always most-recent silently — ignores assignee; stale-employee resume risk.
3. Always show picker if any topic exists — consistent UX but extra clicks for the 1-topic case.
4. Resume by assignee, never picker — same as 1 but pick most-recent silently in ≥2 case.

Eric selected: **Resume by assignee, picker only when ambiguous.** → D-01 in CONTEXT.md.

### Turn 2 — Picker UX shape

Question: *"Picker UX shape — when ≥2 chat topics exist for this issue with the SAME assignee and ambiguity must be resolved, what surface should disambiguate?"*

Options:
1. **Reuse RCB-06 popover, pre-filtered + auto-opened** *(recommended)* — zero new component; implicitly resolves Q4.
2. Route to chat surface; topic-strip is the picker — reuses chat UI; more clicks.
3. Inline popover from button (new component) — duplicates RCB-06's purpose.
4. Mid-screen modal — overkill for 2-3 candidates.

Eric selected: **Reuse RCB-06 popover, pre-filtered + auto-opened.** → D-02 in CONTEXT.md.

### Turn 3 — Cross-employee fall-through

Question: *"Cross-employee fall-through — 0 topics with current assignee but N>0 topics about this issue with other (past) employees. What fires?"*

Options:
1. **Treat as cold — seeded New Topic dialog** *(recommended)* — two clean rules: button = current-assignee only; popover = full history.
2. Auto-open RCB-06 popover with ALL topics — mixes scopes.
3. Seeded New Topic dialog + 'N other topics' link — compromise.
4. Always auto-open RCB-06 popover — even at 0/0; loses friction-free cold-start.

Eric selected: **Treat as cold — seeded New Topic dialog.** → D-03 in CONTEXT.md.

### Turn 4 — Archived topics

Question: *"Archived topics (CTT-07 / Plan 04.1-05) — if a same-assignee topic exists for this issue but is archived, does Continue-in-chat resume it, ignore it, or surface it?"*

Options:
1. Ignore archived — treat as not present *(recommended; respects archive intent)*.
2. Resume archived silently when it's the only match — friction-zero but quiet contradiction.
3. Show in picker, but flagged 'archived' — middle ground.
4. **Auto-unarchive on resume** — turns Continue-in-chat into an implicit un-archive affordance.

Eric selected: **Auto-unarchive on resume** (chose the most aggressive option, deliberately). → D-04 in CONTEXT.md.
