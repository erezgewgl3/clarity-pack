<task type="deferred" tdd="false">
  <name>Task 5 (DEFERRED): Visual regression baseline</name>
  <action>
Visual regression for plugin UI is non-trivial because the plugin renders inside Paperclip's same-origin shell — there's no standalone dev server to point Playwright at. Options:

A. **Static-sketch regression**: Playwright + image snapshot against the four sketch HTML files in `sketches/`. Pinned baseline; PR workflow regenerates + diffs. Catches design-system drift, not actual runtime UI drift.

B. **Live-host regression**: Playwright + Paperclip dev server bootstrap + plugin install + screenshot the four surfaces. Catches real drift but needs Paperclip workspace + database in CI — significant infra.

C. **Storybook + Chromatic**: a third-party visual-regression-as-a-service. Lowest effort but external dependency + cost.

DEFERRED to a follow-up plan (Plan 05-04 or a Phase 5.1) — needs a design choice on which approach. This plan ships lockfile + a11y; visual regression remains the outstanding DIST-05 sub-requirement.
