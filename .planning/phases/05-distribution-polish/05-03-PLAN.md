**A1 — Event source: COMMENT-MARKER.**
Scanner regex (initial grammar, expand only if Eric flags ambiguity):
- Match: `^\s*AC\s*[:\-]\s*(?<id>[A-Za-z0-9_\-]+)\s*[:\-]\s*(?<state>✓|done|complete|x)\s*$` (multiline; case-insensitive on the state)
- Match also: `^\s*AC\[(?<id>[A-Za-z0-9_\-]+)\]\s*[:\-]\s*(?<state>✓|done|complete|x)\s*$` (bracket-style alternate that survives Markdown auto-formatting)
- `id` matches the AC item's `id` field as persisted in the existing AC store (NOT the human label — see `src/worker/handlers/ac-checklist.ts` for the shape). Operators who prefer labels can copy the id from the Reader's AC list (DIST-03 ships a one-click "copy AC marker" button per row).
- Scope: scan all comments on the issue itself; do NOT cross-scan related issues. Single-issue context, single source of truth.

**A2 — UI surface: SIDE-BY-SIDE, manual is source of truth.**
Each AC row keeps its existing manual checkbox unchanged (Phase 2 behaviour structurally untouched). To its right, a small auto-status indicator:
- detected: `auto: ✓ via @<agent-name> · <relative-time>` (small caption-grey text)
- pending (no marker matches this AC id): nothing rendered (no visual noise for the common case)
- The agent-name comes from the comment's `authorAgentId` resolved via `ctx.agents.get` (same pattern Plan 04.2-06 D9 established for `chat.openForIssue`).
- The relative-time uses the existing humanize-time util (`src/ui/util/humanize.ts` or wherever the bulletin uses it — reuse, don't duplicate).

**A3 — Conflict model: NO CONFLICT.**
Manual checkbox state persists to the existing AC store (Phase 2 path unchanged). Auto-status is render-time only — never written. The two live on separate axes and can disagree without resolution. Operator's manual check is always the source of truth.

**A4 — "Copy AC marker" affordance.**
Each AC row gets a small "copy marker" button (clipboard icon) that copies the exact `AC: <id>: ✓` string the scanner expects. Reduces operator discipline cost A1 introduced. NOT load-bearing; ship only if it fits within scope without slipping.

**Scope NOT in this plan (defer to v1.1+):**
- Agent-emitted `clarity.ac.completed` events (the principled deep integration; defer)
- Attachment-derived AC flipping (defer)
- Status-event coarse flipping (defer)
- Notification-on-auto-detect (no notification system at all in v1)
- Conflict UI ("auto says ✓ but manual says ☐") — there is intentionally no conflict to resolve
