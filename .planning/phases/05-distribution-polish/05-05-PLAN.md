<objective>
Ship the three "zero rabbit-holes" finishers that make peek-without-navigation a
real UX option and close the GAP-PICKER-ROW-DISPATCH defect carried from
rc.7: (1) a generic paused-agent banner mounted on BOTH chat header and Reader
top-of-tab with three distinct copies per pause cause (operator / budget /
adapter); (2) a hover-only peek card on every BEAAA-NNN ref-chip showing
title + status + owner display name + first line (≤120 chars) of the
description; (3) a `buildTopicDeepLink` extension that carries
`employeeUserId` so the ambiguous-picker row click lands on the chat surface
with the right employee preselected, plus a full audit of every caller of
that helper to apply the fix consistently.

Purpose: The core value of Clarity Pack is "zero rabbit-holes" (CLAUDE.md).
Three places still force the operator into a rabbit-hole — the paused-agent
state is silent on chat / Reader (operator only finds out by clicking
through to the agent panel); the ref-chip is a click-or-skip affordance with
no glance-able preview, so every cross-reference still costs a navigation;
and the picker row in the rc.7 ambiguous-route popover dispatches an empty
chat surface. This plan closes all three.

Output: One new UI primitive (`agent-pause-banner.tsx`), two extended worker
handlers (`editor-pause-status.ts`, `resolve-refs.ts`) returning richer
typed payloads, one extended ref-chip with hover-peek popover, one extended
`buildTopicDeepLink` + caller audit, two new test files. NO version bump in
this plan — the single phase-wide rc.7 → 1.0.0 bump lives ONLY in Plan 05-10
(closure). Plans 05-04..05-09 ship as code commits + SUMMARY.md without
producing intermediate tgzs.

Wave dependency: this plan is wave 2 and depends on Plan 05-04 because both
plans touch `src/ui/surfaces/reader/index.tsx` (05-04 lands previewer dispatch
~line 330; 05-05 then mounts the banner at top-of-tab on a stable file). The
sequential ordering prevents same-wave file-ownership conflicts.
</objective>
