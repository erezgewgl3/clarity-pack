```
Chat composer [📎 Attach] → file picker (.xlsx | .pdf | .md | .png)

```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → worker | UI passes a base64-encoded file body + declared mime + declared filename to the worker via the SDK bridge. The body is untrusted; the worker MUST sniff and size-guard before invoking `ctx.issues.documents.upsert`. |
| Plugin → host docs API | `ctx.issues.documents.upsert` is the host-trusted boundary; only the worker can call it (declared capabilities `issue.documents.read` + `issue.documents.write` already in manifest). The UI cannot bypass the worker to write documents. |
| chat_message_attachments → chat_messages | Plugin-namespace FK with NOT NULL + ON DELETE CASCADE; referential integrity guaranteed at insert time because chat_messages row commits before any chat_message_attachments insert per Option B upload-on-send. (Plain FK — NOT DEFERRABLE.) |
| Reader Deliverable section ← plugin doc store | `ctx.issues.documents.list` reads from the same plugin-owned store the chat upload writes to; auto-sync is structural (no second protocol). |
