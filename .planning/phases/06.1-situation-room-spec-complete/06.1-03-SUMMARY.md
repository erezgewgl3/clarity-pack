1. **`ArtifactChipRow`** — new per-agent inline chip primitive that REPLACES the Phase 2 bottom `<ArtifactsShippedShelf />` (D-02). The artifacts-shipped-shelf.tsx file is DELETED; the 7 orphaned CSS selectors are DELETED.

### Rule 3 — "real-DOM render tests" maps to source-grep convention (Plan 05-07 D-14)

**Found during:** Task 1 + Task 2 RED test design.

**Issue:** The PLAN.md `<behavior>` blocks describe "real-DOM render tests" with concrete DOM-level assertions (e.g., "queryByTestId(...) returns null", "DOM presence of backdrop element"). The Clarity Pack repo has **NO jsdom**, **NO TSX test transform**, **NO test-renderer** in devDependencies. Every existing UI test in `test/ui/` is a source-grep / static-analysis test reading `.tsx` as text — see `test/ui/chat-react-key-console-capture.test.mjs` Plan 05-07 D-14 deviation block for the canonical reasoning. Adding jsdom + `@testing-library` + a TSX loader would be a new-dep checkpoint, explicitly excluded by CONTEXT.md "NO new runtime dependencies."

**Resolution:** Adopt the same convention as Plan 05-07's D-14 console-capture proxy: every PLAN.md behavior maps onto a source-grep / structural assertion against the source. The 42 tests in this plan collectively pin every invariant the real-DOM tests would assert (terminal classification gates the button cluster; `disabled` attribute is set when `viewerUserId` is null; the dispatch params include all 4 canonical fields; the dialog opens with empty defaults; the blocker context is a sibling not a child of the dialog; gold accent is restricted to Take-Ownership; etc.). The `no-react-key-warnings.test.mjs` static analyser is the load-bearing companion for the "zero React-key warnings" requirement — it is run as part of the broader UI suite and continues to pass.

**Files modified:** `test/ui/surfaces/situation-room/artifact-chip-row.test.mjs` + `test/ui/surfaces/situation-room/critical-path-affordances.test.mjs` adopt source-grep + structural assertions throughout.

### Rule 3 — `critical-path-strip.tsx` extracts a `CriticalPathRow` inner component (no plan deviation; documentation only)

The plan called for adding the button cluster inline within the existing `.map()` block. Each row's button cluster owns local React state (claiming flag + dialog-open flag); inlining state hooks inside a `.map()` violates React's rules-of-hooks. The standard pattern is to extract each iterated row into its own component — `CriticalPathRow` — that owns the per-row state. No behavior change relative to the plan; the structural extraction was implicit in the per-row button + dialog state contract.
