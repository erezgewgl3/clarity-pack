## Operator Selections

All 4 questions answered with the **Recommended** option:

### Q1 — Schema → "Flat upsert — one row per agent"

- **Rationale:** v1.0-friendly; cheapest write path (single ON CONFLICT DO UPDATE); ownership history can be added as additive migration in v1.1+ without breaking the contract.
- **Locked as:** D-01 (CONTEXT.md §Implementation Decisions).
- **Derived implicit:** Migration 0013 ships 3-column table; `company_id` discriminator (4th column) added by Claude's discretion (D-08) to match every other plugin-namespace table's company-scoping pattern.

### Q2 — Artifacts → "Per-agent inline chips, replace bottom shelf"

- **Rationale:** Single source for ROOM-04 + ROOM-10. Tighter agent ↔ artifact coupling. Window default 24h, configurable via `instanceConfigSchema` (bundled into the option description).
- **Locked as:** D-02 + D-05 (the 24h-default is the implicit confirmation embedded in the option label).
- **Derived implicit:** Bottom shelf component is DELETED (not just unmounted); per D-02. ROOM-04 traceability stays "Implemented (Phase 2)" with an appended "UI relocated in Phase 6.1" note.

### Q3 — Create task → "Both — standalone header + per-blocker"

- **Rationale:** Two entry points for two intents — cold task creation (standalone, mirrors other surfaces) vs. converting a known blocker into an assignable task (per-blocker, new affordance). `ClaritySurfaceHeader` already exists (Plan 05-08 D-17); Situation Room joins the mount registry with a 1-line change.
- **Locked as:** D-03.
- **Derived implicit (D-12):** Per-blocker dialog NEVER pre-fills title/description/assignee. Operator typed-fresh; per project memory `feedback_trust-the-clarification-loop` ("the agent+chat+comment loop is designed to ASK for what's missing; front-loading scaffolding works against it"). This is a load-bearing constraint and is locked verbatim from operator preference.

### Q4 — Cycle UI → "Render cycle label as-is, defer auto-break to v1.1+"

- **Rationale:** v1.0 ships; cycles are rare (operator confirmed never seen in rc.8 drill); auto-break heuristic is a v1.1+ design problem worth solving when there's evidence of need.
- **Locked as:** D-04.
- **Derived implicit (D-13):** CYCLE terminal rows get `+ Create task` button only (no Take-Ownership). Take-Ownership presupposes a single leaf agent; CYCLE terminals don't have one. Operator's escape hatch from a cycle is to create a task ("break the A↔B blocking cycle") that they own — a `+ Create task` action.
