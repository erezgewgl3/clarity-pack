## Lessons

### Lesson 1 — Build + parse are two halves of the same contract; round-trip-test the carrier

Plan 06.1-12 (v1) shipped a working `buildChatDeepLink` for `route: 'employee-only'` but didn't extend `parseChatDeepLink` — the parser's guard `if (!newTopic && !topic) return null` silently rejected employee-only payloads, so the chat surface dispatch effect returned early (`link === null`) and the operator landed on the empty-state. The v2 fix added a round-trip test (`06.1-12-deep-link-employee-only.test.mjs` v2 case) that builds an employee-only link, extracts the hash, feeds it to the parser, and asserts the parser returns the expected `ChatDeepLink` object.

**Rule:** any deep-link carrier change MUST ship with a build→encode→decode→parse round-trip test in the same commit. The contract single-source-of-truth design (Plan 04.2-03) only works if both halves are exercised.

### Lesson 2 — COEXIST drills are skip-when-preserved-by-construction

For phases whose only schema delta is additive plugin-namespace, AND whose register drawer explicitly says coexistence is "preserved by construction," AND a prior phase has already passed the disable/enable rowcount drill on the same instance, the COEXIST-NN drill adds zero signal. Pulling live DB credentials via `sudo grep DATABASE_URL` for a redundant rowcount block is wasteful and triggers the auto-mode classifier correctly. Filed as MemPalace `clarity_pack/runbook` drawer `phase-6.1-closure-coexist11-skip` + memory `feedback_coexist-drill-skip-by-construction`.

### Lesson 3 — Query MemPalace BEFORE recommending VPS commands, every time

The operator caught two near-misses this session: (1) recommending `pnpm paperclipai plugin install <tgz>` without first checking MemPalace for the "Missing package.json" gotcha that the README documents and `scripts/install-helper.sh` already solves; (2) recommending a `sudo grep DATABASE_URL` + psql rowcount for COEXIST-11 when the phase register drawer already says "preserved by construction." Both would have been avoided by `mempalace_search wing=clarity_pack room=runbook` as the first action. Reinforces existing memory `feedback_mempalace-before-vps-commands`.
