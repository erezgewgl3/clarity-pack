<age_source_note>
The SDK `Issue` type re-exports from `@paperclipai/shared` (not on disk here). Do NOT assume a specific
"blocked-since" field name. The builder accepts the age input defensively: read the first present of
`issue.updatedAt` / `issue.statusChangedAt` / `issue.blockedAt` / `issue.createdAt` (whatever the live Issue
carries), parse to ms, and compute `age_ms = Date.now() - that`. If NONE parse, set `age_ms = null` and the
UI renders the row WITHOUT an age chip (never renders "NaN" / "Invalid Date"). The TDD test pins both the
present-timestamp path (age computed) and the absent-timestamp path (age_ms null, no throw).
</age_source_note>
