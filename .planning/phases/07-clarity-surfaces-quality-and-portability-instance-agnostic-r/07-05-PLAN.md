<gloss_result_shape_note>
The gloss agent task emits ONE result body = a JSON object `{ "<threadId>": "<one-line gloss>", ... }` (a map
keyed by `LineageThread.id`). `driveBulletinGlossStep` validates it (JSON.parse in a try/catch; non-object or
parse-throw → treat as all-null glosses, never throw), then returns the threads with each `thread.gloss` set
from the map (missing key → null). The cache `body` stores the RAW JSON map string; a cache-hit re-parses it.
The content-hash key is `tldrContentHash({ surface:'bulletin', scopeId:'bulletin-gloss:<cycle>', inputs:{ body:
<canonical JSON of the filtered thread ids+node-detail signatures>, comments:[], refs:[] } })` so the gloss
recompiles ONLY when the surviving thread set changes (a new edition, or the filter output shifts), not on every
view. The gloss PROMPT instructs the Editor-Agent: for each thread (id + its node details), write a single
plain-English sentence ("what this means for you, a busy founder") and return STRICT JSON mapping id→sentence,
no prose around it. Keep the prompt instance-agnostic (no BEAAA literal).
</gloss_result_shape_note>
