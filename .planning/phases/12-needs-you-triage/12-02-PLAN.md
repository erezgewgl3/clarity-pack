<objective>
Make "Needs you" tell the truth and rank by leverage. Two requirements land here, both in the worker triage that feeds the Situation Room snapshot:
- NY-01: Needs-you membership keys STRICTLY off the engine `needsYou` boolean (AWAITING_HUMAN + UNOWNED). agent-working (AWAITING_AGENT_WORKING / in-motion) and self-resolving (SELF_RESOLVING / watch) NEVER appear in Needs-you — the hard invariant D-11. Membership must never use a string match like `ownerName === 'Unassigned'`.
- NY-02: order Needs-you rows by leverage = the count of distinct blocked items whose flattened chain terminates at this action ("items it frees"), descending; tie-break = stable deterministic issue id; NO time/age in the sort (D-01/D-02). Per-leaf dedup: one action item per distinct leaf (D-03). Leverage is computed by reverse-counting the engine-supplied `edges`/`pathIds` (the dependency graph the engine already walked) — NO new host fetch (D-01). Leverage is SORT-ONLY this phase (D-07): do NOT render an "unblocks N" badge or impact text. Leverage ranking applies to the Situation Room Needs-you list ONLY (D-08) — the org-blocked backlog is not re-ordered.
- D-12: the needs-you-banner's single "most-urgent action" (`topAction`) becomes the highest-leverage item (top of the ranked list; stable-id tie-break), not the oldest, so the banner pick and the list-top agree.

Purpose: turn the engine verdict into an honest, leverage-ranked Needs-you list. No new screen, no migration, no new fetch (D-10).
Output: a pure `leverage.ts` helper + an edited `build-employees-rollup.ts` that applies it + tests.
</objective>
