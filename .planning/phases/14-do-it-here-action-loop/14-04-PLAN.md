<objective>
Thread the two signals waves 2-3 depend on — a REAL worker-emitted `needsDurabilityFlip` boolean (BLOCKER 2+4) and the leaf `terminalKind` (BLOCKER 5) — onto the blocker-row data model, and close the OrgBlockedRow field gap (BLOCKER 3) so `<ReplyInPlace>` has everything it needs on all three surfaces. This is the FOUNDATION wave: the predicate (14-02) and the surface wiring (14-03) both read these fields, so they must exist first. Pure additive projection — no engine change, no new host fetch, no schema change.

**The critical correctness rule (BLOCKER 2+4):** `needsDurabilityFlip` is NOT proxied by `terminal.kind`. It is the literal "is the leaf issue currently `status==='blocked'`?" fact, emitted by the worker from data it already has at build time. The Shape-B durable flip (14-01) fires off THIS boolean. No "default false when unknown" on the dominant AWAITING_HUMAN+status=blocked path — both builders have the leaf status in hand.

Purpose: one source of truth for the Shape-B signal (worker-derived from real leaf status), threaded faithfully through every row shape and its UI mirror, so the handler's flip can never be wrong about whether the leaf was blocked.
Output: edits to the two worker builders + the two UI mirror types + two worker tests.

**RISK RESOLUTION (the one flagged risk — CLOSED):** The plan-check flagged "leaf status may not be available for needsDurabilityFlip." It IS available in BOTH builders:
- `build-employees-rollup.ts`: `blockerChain` is only built when `state === 'blocked'` (L334) and `focusIssue.status` is in scope (L296); when the chain is multi-hop the leaf is fetched at L399-406 and the resolved `leaf.status` is available there. So the leaf status = `(resolved leaf).status ?? focusIssue.status`.
- `org-blocked-backlog.ts`: `buildEdges` populates `nodeMeta[toId].status` for every node (L317-323), the leaf node id is `chain.pathIds[last]` (=== `chain.targetIssueUuid`), and the root issue itself is already `status==='blocked'` (the list filter, L349/L357). So the leaf status = `nodeMeta[leafId]?.status ?? 'blocked'`.
No new host fetch is required. If, during execution, a builder genuinely lacks the leaf status for a degraded row, default `needsDurabilityFlip = false` for THAT degraded row only (comment-only is spike-safe) — the dominant happy path always has it.
</objective>
