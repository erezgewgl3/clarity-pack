<objective>
Wave B — make the snapshot honestly degrade-safe and DoS-resistant: run the shared edge-graph build (the irreducible relations.get RPCs) through the bounded pool from 16-01, give every walk a per-call deadline, and floor any slow/hung/thrown walk to the deterministic UNCLASSIFIED line so one stuck node can never block or blank the whole view. Bound the rollup's per-agent fan-out too.

Purpose: SQL-ifying the lists (16-02) removes most round-trips, but the relations edges CANNOT be SQL-ified (no relations table in coreReadTables) — they stay on RPC. An unbounded `Promise.all` over a large roster stampedes the host Postgres, and a single hung `relations.get` would otherwise wait the 30s default → 502. Bounded concurrency + a per-call deadline is BOTH the SNAP-02 degrade-safety fix AND the DoS mitigation (RESEARCH Security Domain).
Output: `mapBounded` + `withDeadline` wired around the shared edge-graph build and the rollup fan-out, with an overall snapshot deadline budget and a degrade-safety test.
</objective>
