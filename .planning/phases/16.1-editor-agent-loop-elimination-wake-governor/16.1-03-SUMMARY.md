# Phase 16.1 Plan 16.1-03: Observe-Only Ingress + Opt-In Scope Gate Summary

The ingress half of the loop fix. Event ingress is now **observe-only** and **opt-in-scoped**: an issue/comment event whose company is not opted in, or that Clarity itself authored, is dropped **before any host call** — and no event handler can reach a wake. Combined with Plan 02's delivery-path severance (`requestWakeup` deleted), the 2026-06-04 wake storm is broken by construction: no Clarity write wakes an agent from the delivery path (Plan 02), and no event handler does host work for an out-of-scope company or reaches `runHeartbeat`/`requestWakeup`/`ctx.agents.*` (this plan).

## What Was Built

- **`opted-in-company-set.ts`** (D-12 / L-2 / L-3 / W-2) — the membership primitive behind the scope gate (LOOP-04). A module-level in-memory `Set<companyId>` seeded **lazily on first handler invocation** (never at boot — L-3) in two pure plugin-namespace `ctx.db.query` SELECTs, **no host call**:
  - (a) opted-in `user_id`s from `clarity_user_prefs WHERE opted_in_at IS NOT NULL` (reuses the opt-in-guard SQL shape).
  - (b) maps those users to companies via the durable `clarity_agent_owners` table (migration 0013): `SELECT DISTINCT company_id ... WHERE owner_user_id = ANY($1)`. This is the **W-2 / L-2 resolution** of the "clarity_user_prefs has no company_id column" landmine.
  - Exports `isCompanyOptedIn` (pure in-memory, zero DB), `ensureSeeded` (lazy/TTL seed), `invalidateOptedInCache`. 60s TTL fast-path (a repeat seed within the window is a no-op). **Fail-closed**: a seed query error leaves the set as-is and logs — never everyone-opted-in. Default state is empty (default OFF, coexistence #1).
- **`worker.ts` ingress (observe-only)** (LOOP-01/02/04, D-02/D-04/D-13) — the `issue.created` / `issue.updated` / `issue.comment.created` handler now: (1) `await ensureSeeded` + `isCompanyOptedIn` scope gate; (2) `isRememberedOwnOperationIssue` fast-path then the authoritative durable `isOwnOperationIssue` provenance check; (3) surviving events are **logged only** — no `enqueue`, no `requestWakeup`, no `runHeartbeat`/`handleEditorHeartbeat`, no `ctx.agents.*`, no `reconcile`. The `HeartbeatDispatcher` is no longer fed from ingress (disposition (b)); its constructor stays in module scope.
- **`worker.ts` company.created** (L-5 #1) — `reconcileEditorAgent` is wrapped by an `isCompanyOptedIn` scope check; a company nobody has opted into never gets a reconcile (unscoped host work eliminated).
- **`chat-stream-bridge.ts`** (L-5 #2) — gains an `isCompanyOptedIn` short-circuit for cost parity. It does SSE re-emit (no wake); the gate runs before the `getChatTopicByIssueId` + `listComments` host work. `ctx.streams.emit` is an SSE publish, not a wake — no forbidden-token false-positive.
- **`set-opt-in.ts`** (D-12) — calls `invalidateOptedInCache()` after the prefs write so an opt-in toggle refreshes the set on the next ingress invocation.
