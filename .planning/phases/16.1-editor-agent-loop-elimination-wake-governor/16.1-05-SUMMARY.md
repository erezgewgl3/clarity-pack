# Phase 16.1 Plan 16.1-05: Falsifiable Loop-Safety CI Gate + Read-Time No-Touch Guard Summary

Made the loop fix falsifiable. Three `test/loop/*.test.mjs` files now form the CI contract that lets Clarity be reinstalled on BEAAA: a region-scoped static gate fails the build the instant a wake token reappears inside an event-handler body (or `requestWakeup` returns to the delivery path); a storm harness reproduces the exact 2026-06-04 incident conditions across a simulated restart and proves wakes are bounded by the governor ceiling with a durable kill-switch; and an opt-in-OFF burst proves zero host work by construction. LOOP-06 is regression-proofed: every read-time test stays green and a phase-wide diff confirms zero edits to the no-touch source list. All three tests pass on the corrected build AND fail when the loop is reintroduced (both falsifiability paths verified by scratch reintroduction + revert).

## What Was Built

- **`test/loop/no-wake-from-ingress.test.mjs`** (LOOP-01, 4 tests) — Layer 1 STATIC, REGION-SCOPED (W-1): strips comments from `src/worker.ts`, extracts the body of each `ctx.events.on(...)` registration via a balanced-brace walk, and asserts each handler body contains ZERO of `requestWakeup / requestWakeups / .pause( / .resume( / managed.reconcile( / runHeartbeat / handleEditorHeartbeat`. The `runHeartbeat`/`handleEditorHeartbeat` tokens are NOT scanned whole-file, so the legitimate module-scope `new HeartbeatDispatcher({ runHeartbeat: ..., })` wiring (worker.ts ~:531) is correctly excluded (T-161-24) — a separate test positively asserts the dispatcher wiring exists AND is outside every handler body. A whole-file grep asserts `agent-task-delivery.ts` has zero `requestWakeup` (D-05). Layer 2 BEHAVIORAL: reconstructs observe-only ingress with the REAL `ensureSeeded`/`isCompanyOptedIn`/`isOwnOperationIssue` primitives and a faithful fake db, feeds a mixed burst (in-scope + own-op + out-of-scope-company), asserts `wakeCalls.length === 0` (alias-proof).
- **`test/loop/storm-safety.test.mjs`** (LOOP-02/03/05, 2 tests) — `makeStormCtx` fake db simulates the three durable tables (provenance `Set`, `wakeLedger` array, version-scoped kill-switch `Map`) off SQL regex; the REAL `checkAndRecordWake` + `recordOwnOperationIssue`/`isOwnOperationIssue` + `isEngaged` run against it. Test A: pre-authors 30 op-issues (durable + in-memory), feeds 90 events, **clears the in-memory fast-path mid-burst (simulated restart)**, feeds 30 re-entrant agent-result events for the same own-op issues (120 total) — asserts zero ingress wakes and zero recursion creates (the durable provenance is authoritative across the restart, D-04). Test B: drives 20 pull-wakes through the governor — asserts actual wakes `<= 6` (ceiling), `killSwitchEngaged === true`, and the durable version-scoped kill-switch **reads engaged after a restart** (D-08, the original onset condition).
- **`test/loop/opt-in-ingress-gate.test.mjs`** (LOOP-04, 1 test) — with the opted-in-company set empty (no opted-in user -> default OFF), feeds a 50-event burst across 5 companies through observe-only ingress; asserts zero host calls (`issues.*`/`agents.*`), zero wakes, and zero `own_operation_issues` reads (the scope gate short-circuits BEFORE the provenance check). Also asserts the lazy seed fired ONCE for the whole burst (60s TTL fast-path) — no per-event DB churn.
- **LOOP-06 no-touch guard** (Task 3, verification) — ran the full read-time regression suite (121 tests, all green) and a phase-wide `git diff --stat 6292e0c..HEAD -- src/` review confirming ZERO edits to every no-touch file.
