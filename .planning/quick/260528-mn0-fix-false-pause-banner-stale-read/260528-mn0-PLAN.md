# Quick Task 260528-mn0 — Fix false "paused by operator" banner (stale-read)

Source: `.planning/NEXT-SESSION-BRIEF.md` §2 (item #1). clarity-pack stays **v1.0.0** (no version bump). Deploy target: BEAAA (AriClaw) via brief §5 stable-path.

## Problem

The generic `AgentPauseBanner` (Reader top + chat header) shows a red "this employee paused by operator — ▶ Resume heartbeat" banner even when the Editor-Agent is **active**, with the wrong name and a 502 on the button. Three root causes:

- **(a) stale heuristic** — `editor.pause-status` sets `paused = (latest editor_agent_failures row).consecutive >= MAX_CONSECUTIVE_FAILURES`. `recordSuccess` (circuit-breaker.ts:112) only zeroes the in-memory counter and writes NO row, so after a real resume + clean compiles the latest failure row still has `consecutive>=3` → banner latches "paused" forever.
- **(b) wrong arg** — `ctx.agents.get(EDITOR_AGENT_KEY, companyId)` is called with the KEY `'editor-agent'`, but the host `id` column is a uuid → `invalid input syntax for type uuid` → caught → `agentName` null → UI shows "this employee".
- **(c) 502** — `agent-pause-banner.tsx` + chat `context-rail.tsx` both call `usePluginAction('agents.resumeHeartbeat')`, but the worker never registers that action → host 502.

## Approach (verified against source 2026-05-28)

### Task 1 — `src/worker/handlers/editor-pause-status.ts` (fixes a + b)
- Widen `EditorPauseStatusCtx.agents` from `Pick<PluginAgentsClient,'get'>` to `Pick<PluginAgentsClient,'get'|'managed'>`.
- Keep the `editor_agent_failures` read for the legacy footer fields (`lastFailureAt`/`reason`) + cause derivation + the fallback heuristic.
- Add an **authoritative status** block: resolve the Editor-Agent UUID via `ctx.agents.managed.reconcile(EDITOR_AGENT_KEY, companyId).agentId`, then `ctx.agents.get(uuid, companyId)`; set `paused = agent.status === 'paused' || agent.pausedAt != null` (pausedAt read defensively via cast — not on the SDK-typed `Agent`), and `agentName = agent.name`.
- Wrap the whole authoritative block in try/catch. On ANY failure (no companyId, no agents client, reconcile/get throws) **fall back to the OLD heuristic** (`consecutive>=MAX`, `agentName=null`) so the handler never gets worse than today.
- Discriminated return unchanged: `{paused:false,...}` or `{paused:true,cause,agentName,lastFailureAt,reason,[detail]}`. When paused-but-no-failure-row, `lastFailureAt/reason` are null and cause defaults to `'operator'`.

### Task 2 — new `src/worker/handlers/agent-resume-heartbeat.ts` + wire in `worker.ts` (fix c)
- `registerAgentResumeHeartbeat(ctx)` using `wrapActionHandler(ctx, 'agents.resumeHeartbeat', fn)` (opt-in gated like every other action).
- Honor an explicit `agentId` param (chat Quick Action passes the chatted employee's uuid); otherwise resolve the Editor-Agent UUID via reconcile (pause-banner passes no agentId).
- `await ctx.agents.resume(agentId, companyId)`; resolve `{ok:true, agentId}` on success.
- **Throw on failure** (reconcile fail / null id / resume throw) so BOTH UI callers' catch fires their graceful-degrade copy (they branch on throw, not on a returned `{error}`).
- Reuses the proven compile-bulletin reconcile→resume path. Capabilities `agents.managed/read/resume` already in the manifest — no manifest/capability/version change.

### Task 3 — `src/ui/primitives/agent-pause-banner.tsx`
- `onResumeClick` passes `{ userId, companyId }` (was `{ companyId }`) so the opt-in-guard-wrapped action passes for opted-in users. `userId` is already in scope from `useResolvedUserId()`. No locked-literal change. **Do NOT touch** `src/ui/surfaces/reader/pause-banner.tsx` (locked literals).

### Task 4 — tests
- Rewrite `test/worker/editor-pause-status.test.mjs`: fake ctx gains `agents.managed.reconcile` + `agents.get` returning `{id,name,status,pausedAt}`.
  - active agent (`status:'idle'`) + stale `consecutive>=3` row → **paused:false** (the regression test).
  - paused agent (`status:'paused'`) → paused:true, agentName resolved, cause from reason.
  - reconcile throws / get throws → fall back to heuristic (consecutive>=3 → paused:true, agentName null).
  - companyId-less / no agents → fallback heuristic (preserves issue-reader-integration contract).
  - back-compat legacy fields present when paused.
- New `test/worker/agent-resume-heartbeat.test.mjs`: explicit agentId path, Editor-Agent-resolution path, resume-throws → handler throws, opted-out → OPT_IN_REQUIRED.
